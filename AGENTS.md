# Jixi Monorepo

## What is Jixi

Jixi is an AI workflow platform. These packages are client SDKs that let browser and server applications execute workflows hosted on the Jixi API. Workflows run AI pipelines server-side; clients send inputs and receive structured events or a final JSON result.

## Package Map

| Package | Purpose | Framework | Depends On |
|---------|---------|-----------|------------|
| `@jixi/js` | Core browser/Node client — wire protocol, SSE, WebSocket audio streaming, auth, errors | None | — |
| `@jixi/react` | React hooks + provider | React 18+ | `@jixi/js` |
| `@jixi/vue` | Vue 3 composables | Vue 3+ | `@jixi/js` |
| `@jixi/svelte` | Svelte stores | Svelte 4+ | `@jixi/js` |
| `@jixi/node` | Server-side client with session token creation | Node.js | `@jixi/js` |

**Critical rule:** Framework wrappers (`react`, `vue`, `svelte`, `node`) never re-implement wire protocol, SSE parsing, token management, or error handling. Those concerns live entirely in `@jixi/js`. If you're adding HTTP logic to a wrapper package, stop and put it in `@jixi/js` instead.

## Commands

Run from repo root (`/Users/robertcornwall/Jixi/jixi/`):

```
npm run build      # turbo build — all packages, ESM + CJS + .d.ts
npm run test       # turbo test — all packages via vitest
npm run typecheck  # turbo typecheck — tsc --noEmit across all packages
npm run dev        # turbo dev — watch mode for all packages
npm run clean      # remove all dist/ directories
```

Run from a package directory (`packages/<name>/`):

```
npm run test         # vitest run (one-shot)
npm run test:watch   # vitest interactive watch mode
npm run typecheck    # tsc --noEmit for this package only
npm run build        # tsup (one-shot)
npm run dev          # tsup --watch
```

Turbo respects the task dependency graph: `build` builds dependencies first; `test` depends on `build` (local). When working on `@jixi/react`, run `npm run build` from root first so `@jixi/js` is compiled.

## TypeScript Conventions

- **Strict mode everywhere** — `tsconfig.base.json` has `strict: true`
- **Target:** ES2020, `lib: ['ES2020', 'DOM']`
- **moduleResolution:** bundler
- **Dual output:** Every package produces ESM (`.js`) + CJS (`.cjs`) + types (`.d.ts`) via tsup
- **Source maps and declaration maps** are enabled
- Engine minimum: Node.js 18.0.0

## Do Not Touch

- `dist/` in any package — generated by tsup; never edit manually
- `node_modules/` — never commit
- `package-lock.json` at root — managed by npm workspaces

## Specs

Full technical design documents in `specs/`:

- `specs/jixi-js-spec.md` — wire protocol, SSE protocol, auth modes, error codes, token management, v1 scope
- `specs/jixi-react-spec.md` — React hook APIs, provider internals, hook state machines, v1 scope

When in doubt about intended behavior, the specs are authoritative. AGENTS.md files summarize what's actionable for code changes; specs explain the reasoning behind design decisions.
